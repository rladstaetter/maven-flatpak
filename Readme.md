# MavenToFlatpak Generator

A zero-dependency Java utility to transform a local Maven repository into a Flatpak-compatible dependency manifest. This tool bridges the gap between Java/JavaFX builds and the sandboxed, network-less environment of `flatpak-builder`.

## 🚀 The Problem

Flatpak builds are sandboxed and cannot access the internet during the build phase. To build a Java application, you must provide a list of every `.jar` and `.pom` file with their remote URLs and SHA-256 checksums so Flatpak can pre-download them into the sandbox.

## ✨ Features

* **Recursive Scanning:** Traverses your `.m2` directory to find all artifacts.
* **Dual Format Support:** Detects output format (**YAML** or **JSON**) via file extension.
* **Architecture Aware:** Adds `only-arches` tags to dependencies to prevent downloading x86_64 natives on ARM64 and vice-versa.
* **Checksum Calculation:** Automatically calculates `sha256` for every file.
* **Metadata Filtering:** Automatically excludes "garbage" files like `resolver-status.properties`, `.lastUpdated`, and `_remote.repositories`.

## 🛠 Usage

### 1. Build your project locally

Run a build to populate a clean temporary local repository. It is best to do this for both architectures if you are targeting both:

```bash
mvn clean install -Dmaven.repo.local=./temp-repo-arm

```

### 2. Compile the Generator

```bash
javac net/ladstatt/apps/MavenToFlatpak.java

```

### 3. Generate the Manifests

Specify the repository path, the output file, and the target architecture.

```bash
# Generate ARM64 YAML
java net.ladstatt.apps.MavenToFlatpak ./temp-repo-arm maven-dependencies-aarch64.yml aarch64

# Generate x86_64 JSON (if preferred)
java net.ladstatt.apps.MavenToFlatpak ./temp-repo-x86 maven-dependencies-x86_64.json x86_64

```

## 📦 JavaFX Flatpak Template

Use this template for your `app-id.yml`. It handles the OpenJDK 25 extension, JavaFX JMODs, and the offline Maven repository.

```yaml
id: org.example.JavaFXApp
runtime: org.freedesktop.Platform
runtime-version: '25.08'
sdk: org.freedesktop.Sdk
sdk-extensions:
  - org.freedesktop.Sdk.Extension.openjdk25
command: AppName
finish-args:
  - --socket=x11        # GUI Access
  - --share=ipc         # Performance
  - --device=all        # Required for Webcam/Hardware access
  - --share=network     # Required only if app needs internet at runtime
  - --filesystem=home   # Access to user files
modules:
  - name: openjdk
    buildsystem: simple
    build-commands:
      - /usr/lib/sdk/openjdk25/install.sh

  - name: app-module
    buildsystem: simple
    build-options:
      append-path: /usr/lib/sdk/openjdk25/bin
      env:
        JAVA_HOME: /usr/lib/sdk/openjdk25
    build-commands:
      # 1. Build using the offline repository provided by the includes below
      - mvn clean package -B -Dmaven.repo.local=./offline-repository --offline -Denforcer.skip=true
      
      # 2. Setup folder structure
      - mkdir -p /app/bin
      
      # 3. Copy Application files (Adjust paths to your project structure)
      - cp -ar target/classes/AppName/lib /app/
      - cp -ar target/classes/AppName/bin/AppName /app/bin/
      
      # 4. Install Icons (Filenames must match App ID)
      - install -Dm644 icons/icon-64.png /app/share/icons/hicolor/64x64/apps/org.example.JavaFXApp.png
      - install -Dm644 icons/icon-256.png /app/share/icons/hicolor/256x256/apps/org.example.JavaFXApp.png
      
      # 5. Desktop and Metainfo
      - install -Dm644 meta/org.example.JavaFXApp.metainfo.xml /app/share/metainfo/org.example.JavaFXApp.metainfo.xml
      - install -Dm644 meta/org.example.JavaFXApp.desktop -t /app/share/applications
    sources:
      - type: git
        url: https://github.com/user/repo.git
        branch: main
        
      # JavaFX JMODs (Architecture Specific)
      - type: archive
        url: https://download2.gluonhq.com/openjfx/25/openjfx-25_linux-aarch64_bin-jmods.zip
        sha256: <HASH_HERE>
        dest: "env/target/javafx-jmods-25"
        only-arches: [aarch64]
      - type: archive
        url: https://download2.gluonhq.com/openjfx/25/openjfx-25_linux-x64_bin-jmods.zip
        sha256: <HASH_HERE>
        dest: "env/target/javafx-jmods-25"
        only-arches: [x86_64]

      # Maven Dependencies (Generated by MavenToFlatpak)
      - !include maven-dependencies-aarch64.yml
      - !include maven-dependencies-x86_64.yml

```

## ⚙️ Technical Details

* **Base URL:** `https://repo.maven.apache.org/maven2/`
* **Target Directory:** All dependencies are mapped to `offline-repository/` within the build sandbox.
* **Requirements:** Java 17 or higher (utilizes `java.util.HexFormat`).
